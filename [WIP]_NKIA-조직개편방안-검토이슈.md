## ISS-04: P10 미안정화의 근본 원인 미분석

### 문제 설명
- B1: "3년+ 개발, 시장 안착 못함"은 현상(Fact)
- 근본 원인: "복합적 (B2+B3+C3)"은 추측

### 분석이 필요한 질문
1. 3년간 무엇을 개발했는가? (기능 범위, 스코프 크립)
2. 경쟁사 대비 개발 속도는? (벤치마크)
3. "시장 안착 못함"의 정의는? (매출? 고객 수? 제품 완성도?)
4. 원인이 조직 문제인가, 기술 문제인가, 시장 타이밍 문제인가?
5. 조직 구조를 바꾸면 이 문제가 해결되는가?

### 해결 방향
- [ ] P10 개발 히스토리 정리 (무엇을, 언제, 왜)
- [ ] "시장 안착 못함"의 구체적 정의 및 근거
- [ ] 근본 원인 분석 (5 Whys 또는 Fishbone)

### 상태
- [ ] 논의 완료
- [ ] 문서 반영 완료

### 분석 진행 중 (2026-01-29)

#### 현상 정리

| 항목 | 내용 |
|------|------|
| P10 상태 | 출시는 했으나, P8 전체 기능 미탑재 |
| 영업 반응 | 기능 부족으로 26년까지 소극적 → 이제 적극 전환 |
| 연구소 입장 | "필요한 기능은 그때그때 대응" |
| 개발 기간 | 3년+ (2022.10 ~ 현재) |

#### 왜 3년이 걸렸나? (잠정 분석)

```
P10 = P8 기능 이전 + 프레임워크 전환 (BE/FE/DB)
      ─────────────────────────────────────────
                    동시 진행
```

| 요인 | 설명 |
|------|------|
| **스코프** | P8 자체가 기능이 많음 → 이전할 양이 많음 |
| **기술 부채** | 프레임워크 전환 (모놀리식→MSA, DB 변경 등) |
| **동시 진행** | 기능 이전 + 기술 전환을 한꺼번에 → 복잡도 폭증 |

#### 복합 원인 구조

```
                    P10 지연
                       │
       ┌───────┬───────┼───────┬───────┐
       ▼       ▼       ▼       ▼       ▼
    스코프    기술    인력    조직    기획
    (D)      (B)     (C)     (A)     (D)
      │        │       │       │       │
   P8 기능   MSA    FE 부족  기획-개발  우선순위
   너무 많음  전환           분리     불명확
```

#### 조직 개편으로 해결 가능 여부

| 원인 | 조직 개편으로 해결 가능? |
|------|------------------------|
| A. 조직 (기획-개발 분리) | ✅ Squad 통합으로 해결 가능 |
| B. 기술 (MSA 복잡성) | ⚠️ 부분적. 기술 거버넌스로 완화 |
| C. 인력 (FE 부족) | ✅ FE 채용으로 해결 가능 |
| D. 기획/스코프 | ⚠️ 부분적. PO 도입으로 개선 가능 |
